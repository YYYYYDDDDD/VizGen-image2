# VizGen 商图引擎使用说明

`VizGen` 是一个纯前端单页工具，入口文件是 `index.html`。它可以把产品图片转换成电商视觉分析报告，再基于报告生成多组生图提示词，最后调用图片编辑接口生成商图。

## 快速开始

1. 用浏览器打开 `index.html`。
2. 在页面顶部配置区填写 `API 地址`、`访问令牌`、`分析模型`、`生图模型`。
3. 点击 `保存配置`。
4. 上传一张或多张产品图片。
5. 按左侧流程依次完成：上传产品图、选择行业/模板/场景、分析并编辑报告、生成并确认提示词、并发生成与下载。

## API 配置怎么填

配置入口在页面顶部，从左到右依次是：

| 页面字段 | 代码 id     | 用途                                  | 示例                       |
| --- |-----------|-------------------------------------|--------------------------|
| `API 地址` | `baseUrl` | API 服务根地址                           | `https://api.openai.com` |
| `访问令牌` | `apiKey`  | API Key / Bearer Token              | `sk-...`                 |
| `分析模型` | `视觉语言模型 VLM`      | 用于产品分析和提示词生成的文本/多模态模型, 只要是多模态的模型都可以 | `gpt-5.4`                |
| `生图模型` | `gpt-image-2` | 用于图片编辑生成的模型                         | `gpt-image-2`            |

**如果没有中转地址，可以联系于老师**

### BaseURL 填写规则

`API 地址` 只填写服务根地址，不要手动加 `/v1`。

正确示例：

```text
https://api.openai.com
```

错误示例：

```text
https://api.openai.com/v1
```

如果使用第三方兼容中转，也同样填写中转根地址，例如：

```text
https://your-api-proxy.example.com
```

## 完整使用流程

### 1. 上传产品图

在第 1 步上传产品图片，支持多张图片。每张图片会成为一个独立产品任务。

可以补充产品信息，例如品牌、规格、目标用户、卖点、禁用词等。补充信息会进入后续产品分析和提示词生成。

### 2. 选择行业、模板和场景

第 2 步用于确定视觉策略：

- `行业`：例如淘宝/天猫、小红书、跨境电商。
- `模板`：不同平台和用途对应不同画面方向。
- `场景`：例如主图、详情页首屏、生活方式场景、促销海报、卖点信息图。
- `额外要求`：写给 AI 的补充约束，比如色调、比例、禁止内容、平台规范等。

### 3. 生成并编辑产品分析报告

这一阶段会根据产品图和上下文生成 Markdown 格式的产品视觉分析报告。报告完成后可以手动编辑，后续提示词会基于这个报告生成。

### 4. 生成并确认提示词

生成多组图片提示词。你可以：

- 勾选或取消某组方案。
- 修改提示词正文。
- 修改图上文案。
- 新增或删除方案。

只有勾选并且提示词非空的方案才会进入最终生图。

### 5. 并发生成和下载图片

每个勾选方案都会提交原始产品图、提示词、尺寸和质量参数。默认最多同时生成 5 张。

生成完成后可以：

- 点击单张图片放大预览。
- 在预览弹层中左右切换多张图。
- 下载单张图片。
- 下载全部成图。

## 常见问题

### 提示“请先填写 API 地址和访问令牌”

检查页面顶部的 `API 地址` 和 `访问令牌` 是否填写，并点击了 `保存配置`。

### 返回 401 或鉴权失败

通常是 Key 错误、Key 过期、Key 没有对应模型权限，或第三方网关不接受 Bearer Token。

### 返回 404

优先检查 BaseURL 是否多写了 `/v1`。本工具会自动拼接 `/v1/responses` 和 `/v1/images/edits`。

### 浏览器报 CORS

说明 API 服务不允许当前网页来源直接访问。解决方式：

- 使用支持浏览器跨域的 API 网关。
- 用后端代理转发请求。
- 本地调试时尽量通过 `http://localhost:8000/index.html` 访问，而不是直接 `file://` 打开。

### 图片生成失败

重点检查：

- 生图模型名称是否正确。
- 服务是否支持 `/v1/images/edits`。
- 上传图片格式和大小是否被接口接受。
- 当前 Key 是否有图片模型权限。
